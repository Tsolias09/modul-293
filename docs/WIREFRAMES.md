# WIREFRAMES – Nikos Skartsolias Portfolio

## 1. Ziel der Wireframes

Die Wireframes zeigen den Aufbau der Website auf Desktop, Tablet und Mobile. Sie dienen als Planungsgrundlage und zeigen, wie Inhalte auf verschiedenen Bildschirmgrössen angeordnet sind.

Die Wireframes sind textbasiert in Markdown erstellt, damit sie direkt im Git-Repository abgelegt werden können.

# 4. Tablet Wireframes

Auf Tablet werden die meisten Bereiche auf 2 Spalten reduziert.

## 4.1 Allgemeiner Tablet-Aufbau

```text
┌───────────────────────────────┐
│ LOGO                 NAV       │
├───────────────────────────────┤
│ Hero / Titel                   │
├───────────────────────────────┤
│ Card        Card               │
│ Card        Card               │
├───────────────────────────────┤
│ Tabellen horizontal scrollbar  │
├───────────────────────────────┤
│ Footer                         │
└───────────────────────────────┘
```

### Tablet-Regeln

- Cards werden meistens zweispaltig dargestellt
- grosse Inhalte werden gestapelt
- Tabellen bleiben scrollbar
- Media Grid wird zweispaltig
- Navigation bleibt sichtbar oder wird reduziert

# 6. Begründung der Wireframes

Die Wireframes wurden so aufgebaut, dass die wichtigsten Informationen immer zuerst sichtbar sind. Auf Desktop wirken grosse Bilder und breite Layouts professionell. Auf Mobile wird alles gestapelt, damit der Inhalt lesbar bleibt.

Besonders wichtig war:

- klare Navigation
- einfache Lesbarkeit
- starke visuelle Wirkung
- gute Darstellung auf allen Geräten
- Fokus auf Spielerinformationen
- einfache Kontaktmöglichkeit

---

# 7. Fazit

Die Wireframes zeigen, dass die Website nicht zufällig aufgebaut wurde, sondern eine klare Struktur besitzt. Jede Seite hat eine eigene Aufgabe und ist für Desktop, Tablet und Mobile geplant.
